🔐 Autenticação (Auth)

POST /auth/login READ-ONLY
Autentica usuário com credenciais do Sankhya e retorna token JWT.

Headers

Nome Tipo Descrição
Content-Type * string application/json
X-Database * string PROD | TESTE | TREINA

Body Parameters

Parâmetro Tipo Descrição
username * string Nome de usuário do Sankhya
password * string Senha do usuário (min 4 caracteres) 
curl -X POST https://api-dbexplorer.gigantao.net/auth/login \
  -H "Content-Type: application/json" \
  -H "X-Database: PROD" \
  -d '{
    "username": "CONVIDADO",
    "password": "guest123"
  }'

Exemplo de Resposta (200 OK)

{
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "tokenType": "Bearer",
    "expiresIn": 3599
  }
}
🔒 Usuário de Teste Disponível: Username: CONVIDADO | Password: guest123
Este usuário funciona nos 3 databases: PROD, TESTE e TREINA
POST /auth/refresh READ-ONLY
Renova o access token usando o refresh token. 
curl -X POST https://api-dbexplorer.gigantao.net/auth/refresh \
  -H "Content-Type: application/json" \
  -H "X-Database: PROD" \
  -d '{
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }'
GET /auth/me 🔐 AUTH
Retorna informações detalhadas do usuário autenticado incluindo dados do funcionário, empresa, gestor e estrutura organizacional. 
curl -X GET https://api-dbexplorer.gigantao.net/auth/me \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"

❤️ Health Check

GET /health

curl -X GET https://api-dbexplorer.gigantao.net/health \

📖 Dicionário de Dados

O dicionário de dados fornece acesso completo à estrutura do Sankhya: tabelas, campos, opções, propriedades e relacionamentos.

GET /dictionary/tables 🔐 AUTH
Lista todas as tabelas cadastradas no dicionário de dados (TDDTAB).

Query Parameters

Parâmetro Tipo Descrição
limit opcional number Limite de registros (padrão: 100)
offset opcional number Offset para paginação (padrão: 0) 
curl -X GET "https://api-dbexplorer.gigantao.net/dictionary/tables?limit=50" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"
GET /dictionary/tables/:tableName/fields 🔐 AUTH
Retorna todos os campos de uma tabela específica (TDDCAM). 
curl -X GET https://api-dbexplorer.gigantao.net/dictionary/tables/TSIUSU/fields \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"
GET /dictionary/search/:term 🔐 AUTH
Busca global no dicionário: pesquisa em tabelas, campos e opções simultaneamente. 
curl -X GET "https://api-dbexplorer.gigantao.net/dictionary/search/PARCEIRO?limit=50" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"

🔍 Inspeção de Tabelas e Queries

GET /inspection/tables 🔐 AUTH
Lista todas as tabelas disponíveis no database. 
curl -X GET https://api-dbexplorer.gigantao.net/inspection/tables \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"
GET /inspection/table-schema 🔐 AUTH
Retorna o schema completo de uma tabela (colunas, tipos, constraints).

Query Parameters

Parâmetro Tipo Descrição
tableName * string Nome da tabela a inspecionar 
curl -X GET "https://api-dbexplorer.gigantao.net/inspection/table-schema?tableName=TSIUSU" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"
POST /inspection/query 🔐 AUTH READ-ONLY
Executa query SQL SELECT (somente leitura). Queries de escrita (INSERT, UPDATE, DELETE) são bloqueadas por segurança.
🔒 SEGURANÇA: Apenas queries SELECT são permitidas. INSERT, UPDATE, DELETE, DROP e outras operações de escrita são bloqueadas automaticamente. 
curl -X POST https://api-dbexplorer.gigantao.net/inspection/query \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -H "X-Database: PROD" \
  -d '{
    "query": "SELECT TOP 10 CODUSU, NOMEUSU, EMAIL FROM TSIUSU"
  }'

📊 Monitoramento SQL Server

Endpoints para monitoramento de performance, queries ativas e estatísticas do SQL Server (requer permissão VIEW SERVER STATE).

GET /monitoring/active-queries 🔐 AUTH
Retorna queries atualmente em execução no SQL Server. 
curl -X GET https://api-dbexplorer.gigantao.net/monitoring/active-queries \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"
GET /monitoring/query-stats 🔐 AUTH
Estatísticas de performance das queries mais executadas.

Query Parameters

Parâmetro Tipo Descrição
limit opcional number Número máximo de queries (padrão: 50, max: 200) 
curl -X GET "https://api-dbexplorer.gigantao.net/monitoring/query-stats?limit=20" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Database: PROD"

🗄️ Seletor de Database

Esta API é multi-database. Selecione qual database deseja usar em suas requisições:

🔒 IMPORTANTE - Política de Escrita:
  • PROD (SANKHYA_PROD): ❌ BLOQUEADO - Somente leitura (SELECT). Nenhuma operação de escrita permitida.
  • TREINA (SANKHYA_TREINA): ❌ BLOQUEADO - Somente leitura (SELECT). Nenhuma operação de escrita permitida.
  • TESTE (SANKHYA_TESTE): ✅ PERMITIDO - Operações de escrita (INSERT, UPDATE, DELETE) são permitidas apenas neste database.

Use o header X-Database: PROD|TESTE|TREINA em todas as requisições para especificar qual database usar.

📝 Notas Importantes

  • 🔐 Endpoints marcados com 🔐 AUTH requerem token JWT no header Authorization: Bearer TOKEN
  • 🗄️ Sempre especifique o database usando o header X-Database: PROD|TESTE|TREINA
  • 📖 Use o Dicionário de Dados para entender a estrutura das tabelas antes de fazer queries
  • 🔍 Use o Inspection para verificar schemas e testar queries com segurança
  • ⚠️ Operações de escrita (INSERT/UPDATE/DELETE) só são permitidas no database TESTE
  • 🚫 Queries maliciosas ou de escrita não autorizada são bloqueadas automaticamente
